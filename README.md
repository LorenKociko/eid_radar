# Ραντεβού RADAR_ (eid_radar)
 
Παρακολουθεί το eid.gov.gr/appointments και κάνει ηχητική ειδοποίηση (beep) μόλις εμφανιστεί διαθέσιμο ραντεβού για **νέα ταυτότητα** μέσα στο επιλεγμένο χρονικό διάστημα. Χρήσιμο επειδή κοντινά ραντεβού βγαίνουν μόνο όταν τα ακυρώνει κάποιος, οπότε το εργαλείο τσεκάρει κάθε λίγα λεπτά και ειδοποιεί για νέα διαθέσιμα ραντεβού.
 
Δεν κλείνει ραντεβού μόνο του, απλά στέλνει ηχητική ειδοποίηση στον browser. Μόλις χτυπήσει, μπαίνεις και το κλείνεις εσύ.

## Οδηγίες
 
**1)** Κάνε login στο `eid.gov.gr/appointments`.
 
**2)** Άνοιξε την **console** (F12 → Console) και τρέξε:
 
```javascript
fetch("https://lorenkociko.com/projects/ID/index.html?v=" + Date.now())
  .then(r => r.text())
  .then(h => { document.open(); document.write(h); document.close(); });
```
 
**3)** Φορτώνει το radar. Διάλεξε περιφέρεια/ενότητα (προεπιλογή ΡΟΔΟΠΗΣ), βάλε πόσες μέρες και κάθε πότε να τσεκάρει, και πάτα **Start watching**. Άσε την καρτέλα ανοιχτή.
 
Το authorization token διαβάζεται **αυτόματα** από τη σελίδα, δεν χρειάζεται να το αντιγράψεις από πουθενά. (Αν για κάποιο λόγο δεν βρεθεί, το radar σου ζητάει να το κάνεις paste χειροκίνητα.)
 
## Ρυθμίσεις
 
| Ρύθμιση | Τι κάνει |
|---|---|
| **Alert window (μέρες)** | Πόσο κοντινό πρέπει να είναι ένα ραντεβού για να χτυπήσει. Π.χ. 5 = ειδοποίηση μόνο για ραντεβού μέσα στις επόμενες 5 μέρες. |
| **Scan every (λεπτά)** | Κάθε πότε τσεκάρει. |
| **Only new slots** | Χτυπάει μία φορά ανά ραντεβού αντί σε κάθε σάρωση. |
| **Stop when found** | Σταματάει στο πρώτο χτύπημα. |

## Απόρρητο

Το εργαλείο δεν αποθηκεύει ούτε αποστέλλει το authorization token, τα στοιχεία σύνδεσης ή τα δεδομένα των ραντεβού σε τρίτους. Όλοι οι έλεγχοι διαθεσιμότητας πραγματοποιούνται απευθείας από τον browser σου προς το `eid.gov.gr`.

Ο κώδικας του radar φορτώνεται από [εδώ](https://github.com/LorenKociko/eid_radar/blob/main/index.html) και εκτελείται τοπικά μέσα στη συγκεκριμένη καρτέλα του browser.

## Γιατί φορτώνεται το radar μέσα στη σελίδα του `eid.gov.gr`;

Το `eid.gov.gr` εφαρμόζει περιορισμούς **CORS (Cross-Origin Resource Sharing)**. Αυτό σημαίνει ότι μια ανεξάρτητη σελίδα που βρίσκεται σε διαφορετικό domain, όπως το `lorenkociko.com`, δεν επιτρέπεται κανονικά να πραγματοποιεί απευθείας αιτήματα προς τα endpoints του `eid.gov.gr`.

Για αυτόν τον λόγο, το HTML του radar φορτώνεται μέσα στην ήδη συνδεδεμένη καρτέλα του `eid.gov.gr`. Με αυτόν τον τρόπο:

- τα αιτήματα πραγματοποιούνται από το ίδιο origin με το `eid.gov.gr`,
- μπορεί να χρησιμοποιηθεί η ήδη ενεργή συνεδρία σύνδεσης,
- αποφεύγονται οι περιορισμοί CORS,
- το authorization token παραμένει μέσα στον browser σου.

Η διαδικασία αντικαθιστά μόνο το περιεχόμενο της συγκεκριμένης καρτέλας. Δεν τροποποιεί μόνιμα τη σελίδα ή τον λογαριασμό σου. Με ένα refresh, η κανονική σελίδα του `eid.gov.gr` επανέρχεται.
